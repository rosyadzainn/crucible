Incident runbook — Database outage v2.3 · pre-hardening
01 ssh -i ~/.ssh/admin_key root@db-primary-01.prod Privileged access without an audit trail. Use sudo with session logging — root over SSH leaves no who/what/when record.
02 psql -c "SELECT * FROM pg_stat_replication" # verify lag < 1s Pre-change verification. Confirms the replica is ready before any failover.
03 systemctl stop postgresql Stops the primary before promoting the replica. Application sees a hard outage window. Promote replica first, then stop primary.
04 tar -czf /tmp/db-snap.tar.gz /var/lib/postgresql /tmp is ephemeral — wiped on reboot. The snapshot is lost the moment recovery requires a restart.
05 psql -c "DROP INDEX users_pkey CASCADE" CASCADE silently drops every dependent object (FKs, views). No backup taken first. One typo = unrecoverable data loss.
06 systemctl start postgresql && nc -z localhost 5432 Port-open ≠ service-healthy. Postgres can listen on 5432 while still rejecting queries. Run a real SELECT 1 probe.
07 pagerduty ack --user oncall-dba --message "primary recovered" Standard handoff. Confirms recovery with the right human in the loop.
